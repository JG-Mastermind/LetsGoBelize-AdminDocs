    Operations Quick Reference Cards¶

    One-Page Laminated Cheat Sheets for Common Tasks

    Print these cards, laminate them, and keep at your operations desk for quick reference during live situations.

    üìã Card 1: Create Admin Invitation¶

    Prerequisites:¶

    • [ ] Super admin role
    • [ ] Recipient's corporate email verified
    • [ ] Role determined (admin, blogger, guide)

    Steps:¶

    1. Dashboard ‚Üí User Management ‚Üí Admin Invitations
    2. Click "Create New Invitation"
    3. Fill form:
    4. Email: [recipient]@letsgobelize.app
    5. Role: [dropdown selection]
    6. Message: Optional welcome note
    7. Click "Send Invitation"
    8. Verify status: "pending"

    48-Hour Rule:¶

    • Invitation expires automatically after 48 hours
    • Cannot extend‚Äîmust create new invitation

    Escalation:¶

    • Email not received: support@letsgobelize.app (1 hour)
    • Link invalid: security@letsgobelize.app (30 min)

    üé´ Card 2: Publish a Tour¶

    Prerequisites:¶

    • [ ] Guide or admin role
    • [ ] Hero image ready (1920x1080px, <5MB, JPG/PNG)
    • [ ] Description written (300+ words)

    Steps:¶

    1. Dashboard ‚Üí My Tours (guide) or Tours Management (admin)
    2. Click "+ Create New Tour"
    3. Fill required fields:
    4. Title (unique, 10-100 chars)
    5. Description (300-2000 chars)
    6. Price (USD, positive number)
    7. Duration (1-24 hours)
    8. Difficulty (easy/moderate/challenging/extreme)
    9. Max participants (1-50)
    10. Upload hero image
    11. Add highlights (4-6 bullet points)
    12. Set availability (days, blackout dates)
    13. Click "Preview" ‚Üí Test mobile view
    14. Click "Publish Tour"
    15. Verify appears on /adventures within 1 minute

    French Translation (Optional):¶

    • Toggle "Add French Translation"
    • Fill title_fr, description_fr, etc.
    • Or click "Auto-Translate" (AI-assisted)

    Escalation:¶

    • Tour not visible: engineering@letsgobelize.app (30 min)
    • Image upload fails: support@letsgobelize.app (1 hour)

    ‚úÖ Card 3: Confirm a Booking¶

    Typical Flow (Automated):¶

    1. Customer completes payment via Stripe
    2. System auto-sets status: pending ‚Üí confirmed
    3. WhatsApp confirmation sent to customer + guide
    4. Admin monitors (no manual action needed)

    Manual Confirmation (Emergency):¶

    Use ONLY if payment succeeded but booking stuck "pending"

    1. Verify payment in Stripe Dashboard:
    2. Search booking ID (e.g., BOOK-2025-12345)
    3. Confirm status: "succeeded"
    4. Dashboard ‚Üí Bookings ‚Üí Find booking
    5. Click "‚ãÆ More" ‚Üí "Mark as Confirmed"
    6. Manually trigger WhatsApp:
    7. Click "Send Confirmation" button
    8. Log incident: "Manual confirmation required due to [reason]"

    Escalation:¶

    • Payment stuck > 30 min: payments@letsgobelize.app (30 min)
    • WhatsApp not sending: support@letsgobelize.app (1 hour)

    üí∞ Card 4: Issue Refund¶

    Refund Policy (Quick Reference):¶

    Cancellation Timing Refund Amount Approver
    > 14 days before tour 100% Auto-approved
    7-14 days before 50% Admin
    2-6 days before 25% Admin
    < 2 days before 0% Super admin
    Guide cancels 100% + $50 credit Auto-approved

    Steps:¶

    1. Calculate refund amount (use policy above)
    2. Stripe Dashboard ‚Üí Payments ‚Üí Search booking ID
    3. Click payment ‚Üí "Refund" button
    4. Enter refund amount (USD, no commas)
    5. Select reason from dropdown
    6. Click "Refund $XXX.XX"
    7. Return to Admin Dashboard
    8. Find booking ‚Üí "‚ãÆ More" ‚Üí "Mark as Refunded"
    9. Enter refund details + notes
    10. Click "Save Refund"
    11. Customer receives auto-email notification

    Limits:¶

    • Admin: Up to $500 per refund
    • Super admin: Unlimited

    Escalation:¶

    • Refund > $500: Super admin approval required
    • Stripe error: payments@letsgobelize.app (1 hour)

    üö® Card 5: Send Safety Alert¶

    Severity Levels:¶

    Level Use Case Channels
    Info Minor disruptions Email only
    Warning Developing situation Email + SMS
    Critical Immediate danger WhatsApp + SMS + Email + Push
    Emergency Active crisis, evacuation All channels + Phone calls

    Steps:¶

    1. VERIFY THREAT (2+ official sources):
    2. [ ] NEMO Belize
    3. [ ] U.S. Embassy alerts
    4. [ ] Belize Met Service
    5. Dashboard ‚Üí Safety Alerts ‚Üí "Create New Alert"
    6. Fill form:
    7. Alert Type: Weather/Security/Health/etc.
    8. Severity: [select from above]
    9. Headline: 5-10 words, urgent
    10. Message: Use template (see Module 06)
    11. Affected Areas: Multi-select locations
    12. Expiration: Date/time when danger passes
    13. Click "Preview Alert" ‚Üí Check mobile view
    14. Click "Send Alert" ‚Üí Confirm
    15. Monitor delivery: Target 95% within 2 minutes

    Message Template:¶

    üö® [ALERT TYPE]: [HEADLINE]

SITUATION: [What's happening, 1-2 sentences]

AFFECTED AREAS: [Locations]

ACTIONS TO TAKE:
- [Action 1]
- [Action 2]

TIMELINE: [When effective, when expires]

CONTACT: [Emergency number]

    Resolution:¶

    When danger passes: - Alert ‚Üí "Mark as Resolved" - Send "All Clear" message

    Escalation:¶

    • Delivery < 80%: support@letsgobelize.app (5 min)
    • Uncertain severity: Super admin + CTO (10 min)
    • Media inquiries: Forward to CTO immediately

    üîí Card 7: Security Incident Response¶

    Recognize Security Incident:¶

    • [ ] Unauthorized login attempts
    • [ ] Role changed without approval
    • [ ] Data access by wrong user
    • [ ] Payment discrepancy
    • [ ] Customer reports fraud
    • [ ] Admin invitation sent to wrong person

    Immediate Actions (5 minutes):¶

    1. DO NOT close browser or logout (preserves audit trail)
    2. Screenshot evidence (include timestamps)
    3. Email security@letsgobelize.app with:
    4. Subject: "SECURITY INCIDENT - [Type]"
    5. Your name, role, timestamp
    6. What happened (3-5 sentences)
    7. Screenshots attached
    8. If user account compromised:
    9. Super admin ‚Üí Disable account immediately
    10. Notify affected users

    DO NOT:¶

    • ‚ùå Try to fix yourself (preserve evidence)
    • ‚ùå Discuss publicly (confidentiality)
    • ‚ùå Delete or modify logs
    • ‚ùå Contact user directly (security team handles)

    Response Times:¶

    • Critical: 15 minutes (security team investigates)
    • High: 1 hour
    • Medium: 4 hours
    • Low: 24 hours

    Follow-Up:¶

    • Participate in incident debrief
    • Complete post-incident report (see template)
    • Update procedures if needed

    üí° Card 9: Common Troubleshooting¶

    Problem: Booking stuck "pending"¶

    Fix: 1. Check Stripe payment status (succeeded/failed) 2. If succeeded > 10 min: Manually confirm (see Card 3) 3. If failed: Booking auto-cancels after 24 hours

    Problem: Tour not visible on /adventures¶

    Fix: 1. Verify is_published = true 2. Check at least 1 future availability date exists 3. Clear browser cache (Ctrl+Shift+R) 4. Wait 2 minutes for search index rebuild

    Problem: WhatsApp not sending¶

    Fix: 1. Check WhatsApp Business API status page 2. Verify message < 1024 characters 3. Retry after 5 minutes 4. If persistent: support@letsgobelize.app

    Problem: Image upload timeout¶

    Fix: 1. Compress image (< 5MB using TinyPNG.com) 2. Convert to JPG (faster than PNG) 3. Try different internet connection 4. If persistent: support@letsgobelize.app

    Problem: "Permission Denied" error¶

    Fix: 1. Verify your role allows this action (see RBAC matrix) 2. Log out + log back in (refresh JWT token) 3. If legitimate action blocked: hello@letsgobelize.app
