    🎯 Learning Paths¶

    New Admin (Week 1-3)¶

    Week 1: Foundations 1. ✅ Read 00-Admin-ReadMe.md 2. ✅ Complete 01-Roles-Permissions-RLS.md 3. ✅ Complete 03-Tours-Management.md 4. ✅ Complete 04-Bookings-Flow-Operations.md 5. ✅ Shadow experienced admin for 1 day

    Week 2: Operations 1. ✅ Complete 05-Payments-Stripe-Connect.md 2. ✅ Complete 06-Safety-Alerts-Runbook.md 3. ✅ Complete 07-WhatsApp-Operations.md 4. ✅ Practice in staging environment

    Week 3: Advanced Topics 1. ✅ Complete 08-AI-Assistant-Governance.md 2. ✅ Complete 10-Analytics-&-Insights-Dashboard.md 3. ✅ Complete 14-Troubleshooting-Playbook.md 4. ✅ Handle real tasks with supervisor oversight

    Blogger Onboarding (Week 1)¶

    Focus: Content Creation Only 1. ✅ Read 00-Admin-ReadMe.md (Sections: Role Matrix, Glossary) 2. ✅ Read 01-Roles-Permissions-RLS.md (Blogger section only) 3. ✅ Read 09-Storage-Images-and-Files.md (blog_images bucket) 4. ✅ Review quick-cards/ops-laminates.md (Card 8: Emergency Contacts)

    Skip: Modules 02-07 (not applicable to blogger role)

    Guide Onboarding (Week 1)¶

    Focus: Tour Management Only 1. ✅ Read 00-Admin-ReadMe.md (Sections: Role Matrix, Glossary) 2. ✅ Read 01-Roles-Permissions-RLS.md (Guide section only) 3. ✅ Complete 03-Tours-Management.md (Full module required) 4. ✅ Read 04-Bookings-Flow-Operations.md (View-only, no refund access) 5. ✅ Read 09-Storage-Images-and-Files.md (tour_images bucket) 6. ✅ Review quick-cards/ops-laminates.md (Card 2: Publish a Tour)

    Skip: Modules 02, 06-08, 11-13 (admin-only features)

    🚀 Using This Manual¶

    For New Admins:¶

    1. Start with 00-Admin-ReadMe.md
    2. Follow the learning path for your role
    3. Complete knowledge checks at end of each module
    4. Ask supervisor to verify competency checklist

    For Supervisors:¶

    1. Assign modules based on role (see Learning Paths above)
    2. Schedule weekly check-ins during training
    3. Use knowledge checks to assess readiness
    4. Sign off on certification checklist before solo operations

    For Incident Response:¶

    1. Use checklists/incident-response.md immediately
    2. Reference mermaid-backend-flows.md to understand affected systems
    3. Complete templates/post-incident-report.md within 48 hours

    For Daily Operations:¶

    1. Keep quick-cards/ops-laminates.md printed at desk
    2. Run checklists/month-end-close.md on 1st of month
    3. Reference 14-Troubleshooting-Playbook.md for common issues
